Module 1: Platform Overview And Navigation

1. Layout Overview

Top Bar

  • Events Dropdown: Displays the currently active event. You can switch between events from this menu.
  • Search Bar: Used to find specific addresses or locations on the map. Type any address to quickly navigate there.
  • View Selector:
    • Map: Displays the main interactive map view.
    • Control Room: Alternative view for monitoring.
  • User & Notifications (top-right):
    • Notification bell shows new notifications.
    • Settings icon opens user preferences and system configuration.

2. Left Sidebar Menu

The left menu provides access to all operational layers of the system:

MenuDescription
InfoDisplays general information of event details.
LayersControls visibility of map elements (groups, structures, incidents, tasks, areas, evacuation routes).
TasksLists and manages all created tasks. You can view, edit, or add new ones.
IncidentsShows current and past incidents on the map. Each incident is represented by a marker.
UnitsDisplays available units (users) and their status.
MessagesOpens the messaging panel for communication.
EvacuationProvides tool for creating or managing evacuation routes.

3. Map Elements

  • Sync button: Import all cameras from FlexControl Registry.
  • Monitor button: Copies map to clipboard.
  • Home button: Centers the event on the map.
  • Ruler button: Measure distance on the map.
  • Zoom in/out buttons: Zoom the map.
Module 2: Plan and Manage A Large Outdoor Event

Plan Event

Create Operation (Event) from Scratch

Accessing Event Creation:

1
From the main navigation, go to Platform Settings → Events.
2
Click the New Item button located at the top of the Events list.
3
The Event Creation screen will open.

Event Details (Left Panel):

Use the panel on the left side of the screen to define the core details of your event:

  • Event Name – Enter a clear, descriptive title for your event.
  • Description – Provide any relevant details, objectives, or notes.
  • Start and End Date – Set the time frame during which the event will be active.
  • Groups – Assign existing groups that will participate in or oversee the event.
  • Users – Add individual users who will take part or have monitoring responsibilities.
  • Devices – Include devices (e.g., GPS trackers, drones) that will be used during the event.

Map Tools:

The right side of the screen contains an interactive map where you can define the geographical aspects of your event:

  • Draw Sectors – Define specific areas where activities or monitoring will take place.
  • Draw Lines – Mark paths or patrol routes relevant to event operations.
  • Place Points – Pinpoint specific locations of interest such as checkpoints or observation posts.
Result: Once all details and map elements are configured, click Save to finalize event creation. Your new event will appear in the Events list.

Create Template from Scratch

Event templates allow you to predefine and store event configurations that can be reused later to create new events quickly and consistently.

1
From the main navigation, go to Platform Settings → Events.
2
Open the Event Templates tab at the top of the page.
3
Click the New Item button to start creating a new template.

Template Configuration:

  • Template Name – Enter a descriptive name for the template.
  • Description – Add notes or context about the template's purpose.
  • Groups – Add one or more groups that will be linked to this template.
  • On the map area, you can add Points, Lines, and Sectors.

Create a Structure on the Map

A Structure represents a fixed location or object on the event map — such as a command post, observation point, or an area equipped with devices like drones or beacons.

1
Navigate to the Event Creation screen or open an existing event in Edit Mode.
2
On the map toolbar, click the GPS icon to activate structure placement mode.
3
Click on the desired location on the map where you want to place the structure.
4
Configure: Icon Selection, Structure Name, and attach devices if needed.
5
Click anywhere on the map to save the structure.

Draw a Sector on the Map

A Sector represents a defined area on the map — such as a patrol zone, restricted area, or operational region.

1
On the map toolbar, click the Sector icon to activate sector drawing mode.
2
Start drawing the area by clicking to create each corner point.
3
Click on the last point to close the area.
4
Configure: Color, Title, Smoothness, and Fill Opacity.
5
Click anywhere on the map to save the sector.

Create Tasks

Tasks are assignments that users must complete during an event. Mapli supports multiple task types for different operational needs.

Location Task

Requires users to reach a specific point on the map within a defined radius.

Area Task

Requires users to monitor or secure a defined zone/sector.

Path Task

Requires users to follow a route through predefined checkpoints.

Action Task

Non-location-based task for general actions or duties.

Beacon Task

Linked to beacon devices for indoor location tracking.

Create a Location Task

1
Navigate to your Event and click Tasks on the left panel.
2
Click Create New Task and select Location Task.
3
Click on the map to place the task location marker.
4
Fill in: Task Name, Description, Start/End Date, Location Radius (min 10m), and Assigned Users.
5
Click Save to create the task.
Click to expand

Create an Area Task

2
Click Draw to create a new area or Select to choose an existing area.
3
Fill in: Task Title, Description, Start/End Date, Group of Users, and Unit count.
4
Click Register the Task to finalize.

Evacuation Routes Planning

The Evacuation feature allows event administrators to plan, visualize, and manage evacuation routes directly on the event map.

Accessing the Evacuation Panel

1
Navigate to your Event.
2
On the left side panel, click the Evacuation button.
3
The Evacuation panel will open with options to turn mode on/off, add routes, or edit existing routes.

Turning Evacuation Mode On or Off

  • Toggle the Evacuation Mode switch in the panel.
  • When activated: notifications are sent to all mobile users, a banner appears in the web interface, and the action is logged.

Adding an Evacuation Route

1
Click Add Route in the Evacuation panel.
2
On the map, click to set the starting point, then click again to set the endpoint.
3
The route will display length and estimated travel time.
4
Repeat for additional routes, then click Save All.

Editing Routes

Each route can be edited directly from the panel. Modify the path or change the mode of travel (Walking, Cycling, or Driving).

Managing Incidents

The Incidents feature allows you to create, view, and manage incidents directly on the event map.

Creating a New Incident

1
Navigate to your Event and click Incidents on the left panel.
2
Click Create New Incident.
3
Click on the map where the incident occurred.
4
Fill in: Incident Level (Critical/Major/Minor), Description, Assigned Groups.
5
Optionally check "Create as Task" to generate a related task.
6
Click Register New Incident.

Managing Existing Incidents

  • Click on any incident in the list or map to edit details.
  • Statuses: Not Resolved (active) or Resolved (addressed).
  • Options: Delete, Locate on Map, Mark as Resolved.

Incident Messaging and Media Sharing

Each incident includes a Comments section for direct communication:

  • Send text messages
  • Attach and send photos
  • Attach and send video files

Activity Log & Notifications

The Activity Log keeps users informed about all important events. Notifications appear as a bell icon with a red counter.

Notification Triggers:

  • New invitation code created
  • User added to a chat channel
  • Channel name changed
  • Task reminders
  • Timeline path imported
  • Request to start video streaming
  • Incident created
  • Message sent in any channel
  • Video streaming started/ended
  • Task user failed
  • User check-in completed
  • Task started with insufficient users
Monitor Real-Time Locations

The Real-Time Location Monitoring feature allows administrators to continuously track the movement of all active mobile users.

How it works: When a user opens the mobile app (even in background), their location is automatically updated on the web map. Location is refreshed every second (or faster) for smooth, accurate movement tracking.

This feature helps event administrators:

  • Maintain full situational awareness
  • Monitor patrolling activities
  • Respond quickly to incidents based on precise, live user locations

Task Status Tabs

Completed

Failed

Searching for Tasks

Use the Search Bar at the top of the Tasks page to search by task title, type, ID, group, user, or any related keyword.

Video Streams

Streaming is available for mobile device users as a way of communicating with web app operators - a live transmission of footage from the users’ camera to the computer screen of an operator. A Web operator must make a request that a specific user would start streaming, but a user can also start transmitting a stream on their own.

Requesting a Stream

1
Open a Units menu.
2
Request a stream directly from the users list by clicking the stream icon next to the specific user.
3
Or select a user in the list and request a stream by clicking the stream icon in the user information menu.
Result: After requesting a stream from the web app, a user with the mobile app gets a notification about the request.

Timeline

The Timeline feature allows event administrators to review user movement history on the map, visualizing tracked paths over a selected time period.

Accessing the Timeline

1
Navigate to your Event.
2
At the bottom center of the screen, click Show Timeline.
3
The Timeline panel will appear with time range and filtering options.

Using the Timeline

  • The system tracks users' locations through the mobile app, drawing paths on the map.
  • Filter by date and time to focus on specific activity periods.
  • The corresponding movement path will appear as a drawn route.

Adjusting the Time Range

Use the orange time bar at the bottom of the panel:

  • Drag edges – Shrink or expand the time frame.
  • Move the bar – Shift the visible time range left or right.
  • The map updates automatically to show movements within the selected window.

Mobile App - Execute Event (Field Personnel)

Field personnel use the mobile app to execute tasks, report incidents, and communicate with administrators in real-time.

Location Change Task

Requires the user to reach a specific point on the map within the defined radius.

Accessing the Task

1
Open the Mobile App.
2
Tap Tasks in the bottom navigation bar.
3
Find and tap your Location Change Task in "My Tasks".

Task Details Screen

  • Navigation Button – Returns to the main map screen.
  • Map Preview – Shows the exact task location.
  • Task Overview – ID, type, and participant info.
  • Task Timing – Start and end times.
  • User Status – Pending, Completed, or Failed lists.
  • Comments Button – Add text, photos, or videos.

Completing the Task

Move physically to the pinned location. Once you enter the defined radius, the system automatically marks the task as Completed.

Click to expand

Area (Sector Monitoring) Task

Requires users to enter and remain within a defined area for the specified duration.

Key Actions

  • Check In – Tap when entering the area to start patrolling.
  • Check Out – Tap when finished.
Important: After checking out, you cannot check in again for the same task. Checking out early marks the task as Failed for your user.
Click to expand

Path Task

Requires users to move through predefined checkpoints on the map.

Completing the Task

  • Locate checkpoints on the map.
  • Move physically toward each checkpoint.
  • When your position enters the checkpoint's radius, it's marked as passed.
  • You can visit checkpoints in any order.
  • Once all checkpoints are passed, the task is Completed.
Click to expand

Action Task

Non-location-based task that can be completed manually.

Completing the Task

Simply tap the Complete Task button to mark it as done.

Note: If an end time is set and the task isn't completed before that time, it automatically fails. Once failed, users cannot manually complete it.
Click to expand

Beacon Task

Requires the user to reach a specific indoor location with a beacon device.

Completing the Task

1
Go to the task location shown on the map.
2
Tap Start Scanning when near the beacon.
3
The system detects your proximity and marks the task as Completed.
Tip: Ensure Bluetooth is enabled on your mobile device to detect beacons.
Mobile App - Creating Incidents

The mobile app allows users to quickly report emergencies using the SOS button.

Accessing the SOS Button

On the main map screen, look for the SOS button in the bottom-left corner.

Creating an Incident via SOS

1
Press and hold the SOS button for 2 seconds.
2
The system automatically creates a Critical incident at your current GPS position.
Result: A Critical incident with status "Not Resolved" is created immediately at your location — no additional input required.

Viewing and Managing Incidents

1
Tap Incidents in the bottom navigation bar.
2
View the list of all incidents related to the event.
3
Tap any incident to view details and add comments.

Commenting & Attachments

Inside any incident, you can:

  • Send text messages
  • Attach photos and videos
  • Record and send voice messages

Mobile App - Evacuation Mode

Evacuation Mode guides mobile users safely to exit points during emergencies.

Receiving Evacuation Alerts

  • When Evacuation Mode is activated in the Web App, all mobile users receive a real-time notification.
  • A red evacuation button with a running person icon appears on screen.

Selecting Evacuation Route

1
Tap the Evacuation Button.
2
Choose your mode: Walking, Cycling, or Driving.
3
Tap Generate to calculate your route.

Route Guidance

  • The system generates a path to the closest evacuation endpoint.
  • Users are always routed to the nearest safe exit based on current location.
  • The path is displayed on the map for users to follow safely.
